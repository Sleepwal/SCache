## 淘汰算法
缓存满了之后，使用**LRU（最近最少使用）算法**淘汰
- 相对于仅考虑时间因素的 FIFO 和仅考虑访问频率的 LFU，LRU 算法可以认为是相对平衡的一种淘汰算法。
- LRU 认为，如果数据最近被访问过，那么将来被访问的概率也会更高。
- LRU 算法的实现非常简单：
  - 维护一个队列，如果某条记录被访问了，则移动到队尾。
  - 那么队首则是最近最少访问的数据，淘汰该条记录即可。
